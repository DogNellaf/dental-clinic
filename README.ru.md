# DentalClinic — Система управления стоматологической клиникой

> [English version](README.md)

Веб-приложение для управления записями на приёмы, персоналом, услугами и отзывами пациентов стоматологической сети. Построено на ASP.NET Core 8 MVC и Entity Framework Core.

---

## Возможности

| Роль | Функции |
|------|---------|
| **Клиент** | Запись на приём, просмотр больничной карты, публикация и скрытие отзыва |
| **Врач** | Текущий и все приёмы, добавление рекомендаций, продление или досрочное завершение приёма, просмотр карты пациента |
| **Менеджер** | Модерация и публикация отзывов пациентов |
| **Администратор** | Полное управление профилями, приёмами и отзывами |

**Публичные страницы** (без авторизации): главная с отзывами, каталог услуг, расписание, FAQ, о клинике, контакты.

---

## Технологии

- **Платформа**: .NET 8.0
- **Фреймворк**: ASP.NET Core MVC
- **ORM**: Entity Framework Core 8 с SQL Server
- **Аутентификация**: ASP.NET Core Identity (cookie)
- **Frontend**: Bootstrap 5, jQuery, jQuery Validation
- **Тесты**: xUnit, Moq, EF Core In-Memory

## Запуск

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd Программа
```

### 2. Настройка строки подключения

Отредактируйте `appsettings.json`:

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=.\\SQLEXPRESS;Database=dental_clinic;Trusted_Connection=True;Encrypt=False;"
  }
}
```

### 3. Заполнение ролей

Приложение использует четыре роли с фиксированными идентификаторами:

| ID | Название |
|----|----------|
| 1 | Клиент |
| 2 | Администратор |
| 3 | Менеджер |
| 4 | Доктор |

После первого запуска (приложение создаст схему БД автоматически) выполните SQL:

```sql
INSERT INTO [Role] (Id, Name, NormalizedName, ConcurrencyStamp, Title)
VALUES
  (1, N'Клиент',        N'КЛИЕНТ',        NEWID(), N'Клиент'),
  (2, N'Администратор', N'АДМИНИСТРАТОР',  NEWID(), N'Администратор'),
  (3, N'Менеджер',      N'МЕНЕДЖЕР',       NEWID(), N'Менеджер'),
  (4, N'Доктор',        N'ДОКТОР',         NEWID(), N'Доктор');
```

### 4. Запуск

```bash
dotnet run
```

Приложение будет доступно по адресу `http://localhost:5137`.

## Роли пользователей

### Клиент (`/client`)
- Просмотр предстоящих и прошедших приёмов
- Подробная информация о приёме и рекомендации врача
- Публикация или обновление отзыва (требуется хотя бы один прошедший приём)

### Врач (`/doctor`)
- Текущий активный приём в реальном времени
- Добавление рекомендаций в запись
- Продление приёма или досрочное завершение
- Просмотр полной карты любого пациента

### Менеджер (`/manager`)
- Очередь отзывов на модерацию
- Публикация или скрытие отзывов

### Администратор (`/admin`)
- Создание профилей с назначением роли
- Блокировка и разблокировка аккаунтов
- Редактирование и удаление приёмов
- Модерация отзывов

---

## Процесс записи на приём

1. Перейдите в раздел **Расписание** и выберите услугу.
2. Выберите врача из списка специалистов по данной услуге.
3. Выберите свободный временной слот.
4. Слот сразу закрепляется за вашим аккаунтом.
5. Просмотрите запись в разделе **Личный кабинет**.
